# Phase 4 — Parallel Testing

## Goal

Run both the old GAS app and the new Flask app simultaneously. Have Larry (and optionally other owners) test all operations on the Flask version and verify that data outcomes match what the GAS app would produce. No users are pointed to the new app yet — this is internal validation.

## Phase 3 Build Notes (for testers)

The following was built and tested in Phase 3. These notes flag differences from the original GAS behavior that testers need to be aware of.

### What was built
- `/calendar/<year>` — Year grid, 9 owner rows × 5 week columns, site_config rows at top
- `/lookup` — Owner/year picker, GET params `?owner_id=N&year=YYYY` (0 = all owners)
- `/form` — Trade/comment form with owner dropdown + JS-populated week dropdown
- `/ics/download` — POST, generates `.ics` using `icalendar` library
- `/email` — Gmail/Outlook Web/mailto compose URLs + clipboard copy
- `/admin/` — Table list; `/admin/table/<name>` full browser; add/edit/delete/export per-table
- `/auth/` — Magic link login; SMTP configured in `.env`, falls back to flash message if not set

### Known differences from GAS behavior

**9 owners, not 10.** The database has 9 active owners (IDs 1–9). Any test referencing "all 10 owners" should be read as "all 9 owners."

**"Give Away" is comment-only.** In the GAS app, Give Away was treated the same as Trade Week (owner2 fields shown, `current_holder_id` transferred). In this version, Give Away only shows owner1 fields and updates the `comment` field — the same as Not Using and Comment. T5 below reflects this. If the original behavior (transferring ownership) is needed, use "Trade Week" instead.

**Previous/Next year links go relative to today, not the viewed year.** `/calendar/prev` and `/calendar/next` redirect to `today.year ± 1`, not `viewed_year ± 1`. The year picker dropdown is the correct tool for navigating non-current years.

**SMTP is not configured yet.** Until `SMTP_HOST` is set in `.env`, magic link URLs appear as flash messages on the login page instead of being emailed. T3 SMTP tests require SMTP setup first.

**Dev test trades in database.** During Phase 3 development, two test trades were made against the live database: Boone 01/01/2026 ↔ Stalker 02/26/2026. **Re-seed `trade_detail` before parallel testing** (see Data Sync below) to restore clean state.

**Comment is truncated server-side to 40 chars.** The form has `maxlength="40"` but the route also hard-truncates — no explicit error message for oversized comments submitted via POST directly.

**Admin table browser shows 9 owners.** The T10 test "All 10 owners listed" should read "All 9 owners listed."

## Data Sync Before Testing

**Required before first parallel test session** — dev test trades were written to the database during Phase 3 development:
- Boone 01/01/2026 ↔ Stalker 02/26/2026 (test trade, should be reverted)
- Kamons 01/08/2026 (test comment added)

**Reseed steps:**
1. Delete `instance/snowbound.db`
2. Re-run the seeding scripts in order:
   ```
   python -m snowbound.scripts.seed_owners
   python -m snowbound.scripts.generate_rotation
   python -m snowbound.scripts.import_trade_detail
   python -m snowbound.scripts.import_audit
   python -m snowbound.scripts.verify_counts
   ```
3. Verify row counts match the Google Sheet

**Before each subsequent test session** (if the GAS app has had activity since last sync):
1. Export updated `TradeDetail` sheet to CSV
2. Re-run `scripts/import_trade_detail.py` (truncate + reload)
3. Verify row counts still match
